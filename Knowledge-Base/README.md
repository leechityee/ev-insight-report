## 🔬 02-方法论

### PA 流程智能体框架

**6 阶段 12 原子活动**:

```
AA-01 意图识别 → AA-02 搜索执行 → AA-03 内容抓取 → AA-04 数据整理
                                                      ↓
         AA-12 报告 ← AA-11 图表 ← AA-10 机会 ←─ 并行分析 (AA-05~09)
```

### 三种工作模式

| 模式 | 交互次数 | 执行时间 | 适用场景 |
|------|----------|----------|----------|
| Model 0 全自动 | 1-2 次 | 2-3 分钟 | 快速预览 |
| Model 1 关键确认 | 3-4 次 | 5-10 分钟 | 常规报告（推荐） |
| Model 2 逐步确认 | 6-8 次 | 15-30 分钟 | 深度研究 |

### 分析维度框架

**10 大核心维度**:
1. 定价与产品矩阵
2. 动力与性能
3. 续航与补能
4. 智能驾驶
5. 智能座舱与生态
6. 底盘与安全
7. 用户画像
8. 销量与市场

## 📚 06-最佳实践

### 报告生成最佳实践

1. **数据准确性**
   - 优先使用官方数据
   - 交叉验证第三方来源
   - 标注数据日期和来源

2. **可视化设计**
   - 移动端优先
   - 关键数据突出显示
   - 避免信息过载

3. **交付方式**
   - 提供多种格式（HTML/Markdown）
   - 支持定时任务
   - 多渠道交付（邮件/飞书/GitHub）

4. **用户体验**
   - 执行前告知模式选择
   - 关键节点确认
   - 交付后询问定时需求

### 常见问题处理

| 问题 | 解决方案 |
|------|----------|
| web_search 不可用 | 降级为 AI 知识库 + web_fetch |
| 中文文件名 CDN 问题 | URL 编码或使用 index.html |
| GitHub Pages 404 | 手动启用 Pages 设置 |
| 移动端表格显示 | 横向滚动或卡片化 |
